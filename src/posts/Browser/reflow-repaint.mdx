## Reflow

<aside>
💡 **Reflow란?**

생성된 DOM 노드의 레이아웃 수치(너비, 높이, 위치 등) 변경 시 영향 받은 모든 노드의 수치를 **다시 계산하는 과정**

</aside>

- DOM Element 추가, 제거 또는 변경
- CSS 스타일 추가, 제거 또는 변경
- CSS3 animation과 transition의 모든 프레임
- offsetWidth와 offsetHeight 사용 → 해당 속성을 읽으면 초기 reflow가 트리거되어 수치가 계산됨
- 유저 인터랙션으로 발생하는 hover, 텍스트 입력, 창 크기 조정 등

## Repaint

<aside>
💡 **Repaint란?**

Reflow 과정이 끝난 후 재생성된 **렌더 트리를 다시 그리는 과정**

</aside>

- 가시성이 변경되는 순간 (opacity, background-color, visibility, outline)
- Reflow가 실행된 순간 뒤에 실행됨

## Reflow, Repaint 최소화 하기

- `transform`, `opacity` 사용 : 두 속성은 reflow와 repaint가 일어나지 않음
- `display: none`일 경우 reflow와 repaint가 일어나지 않음
  - 많은 수의 Element를 변경해야 할 경우, 숨겨진 상태에서 변경하기
