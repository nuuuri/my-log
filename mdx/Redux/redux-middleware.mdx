- action이 dispatch 되어서 reducer에서 이를 처리하기 전에 사전에 지정된 작업들을 설정해줌
- action을 console에 기록, 전달 받은 action에 기반하여 action 취소, 다른 종류의 action을 추가 등등…

```tsx
const configureStore = () => {
  const middlewares = [];

  const enhancer =
    process.env.NODE_ENV === 'production'
      ? compose(applyMiddleware(...middlewares))
      : composeWithDevTools(applyMiddleware(...middlewares));

  const store = createStore(rootReducer, enhancer);

  return store;
};
```

- **`redux-thunk`** : 비동기 작업 미들웨어. 사용하면 action 객체가 아닌 함수를 dispatch 할 수 있음 → 한번에 dispatch를 여러번 할 수 있게 해줌
- **`redux-saga`** : 비동지 작업 미들웨어. action을 모니터링 하고 있다가, 특정 action이 발생하면 미리 정해둔 로직에 따라 특정 작업이 이루어짐
